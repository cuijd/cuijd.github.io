redis的简要总结

针对框架进行redis的一些简要总结

##
Redis 是完全开源免费的，遵守BSD协议，是一个高性能的key-value数据库。
Redis 与其他 key - value 缓存产品有以下三个特点：
Redis支持数据的持久化，可以将内存中的数据保存在磁盘中，重启的时候可以再次加载进行使用。
Redis不仅仅支持简单的key-value类型的数据，同时还提供list，set，zset，hash等数据结构的存储。
Redis支持数据的备份，即master-slave模式的数据备份
Redis支持五种数据类型：string（字符串），hash（哈希），list（列表），set（集合）及zset(sorted set：有序集合)
本质上redis可以通过自己的客户端（类似于mysql和oracle一样）的命令实现各种数据的录入和取出。
甚至包括事物也可以,Redis 发布订阅也可以实现。
redis也像数据库一样在使用时需要建立和redis的链接，所以同理他也有自己的JedisPool来管理jedis连接
在java中使用redis和JedisPool需要添加jedis jar包或redis jar包

redis是一个Key-Value的NoSQL数据库，开发维护很活跃，虽然它是一个Key-Value数据库存储系统，但它本身支持MQ功能，
所以完全可以当做一个轻量级的队列服务来使用。对于RabbitMQ和Redis的入队和出队操作，各执行100万次，
每10万次记录一次执行时间。测试数据分为128Bytes、512Bytes、1K和10K四个不同大小的数据。实验表明：入队时，
当数据比较小时Redis的性能要高于RabbitMQ，而如果数据大小超过了10K，Redis则慢的无法忍受；出队时，
无论数据大小，Redis都表现出非常好的性能，而RabbitMQ的出队性能则远低于Redis。
